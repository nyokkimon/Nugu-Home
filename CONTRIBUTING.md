#### 1. Reporting Bugs
- Check the **Issues** tab to see if the bug has already been reported.
- If not, open a new issue. Include your OS (Linux/Windows), your Python version, and a clear description of the error.
- If possible, include the terminal output or logs (but **hide your MAC addresses and Tokens!**).

#### 2. Suggesting Enhancements
- We are always looking for new features (e.g., better vendor icons, mobile notifications, or Web UI).
- Open an issue with the tag `enhancement` to discuss your idea before writing code.

#### 3. Pull Requests (Code Changes)
- **Fork** the repository and create your branch from `main`.
- Ensure your code follows the existing style (clean, commented, and scannable).
- If you've added a new command, please update the `README.md` and `!nugu` help menu as well.
- Open a **Pull Request** with a clear description of what you changed and why.
